## Site frequency spectrum (SFS)

The site frequency spectrum describes the distribution of allele frequencies across SNP sites. To compute it per site, use `--persite` with the `daf` (derived allele frequency) statistic.

### Per-site DAF with missingness filter

```console
$ cd piawka/examples

# Compute per-site DAF and missingness for all groups
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s daf,miss --persite \
  > persite.bed

# Keep only sites with no missing data, then inspect DAF distribution
$ piawka filt -e 'miss == 0' persite.bed \
  | awk '$7=="daf"' \
  | awk '$5=="LE_2n"' \
  | sort -k8,8n | head -6 | column -t
scaffold_1  10000112  10000113  .  LE_2n  .  daf  0.066667  1   15
scaffold_1  10000130  10000131  .  LE_2n  .  daf  0.200000  3   15
scaffold_1  10000199  10000200  .  LE_2n  .  daf  0.200000  3   15
scaffold_1  10000209  10000210  .  LE_2n  .  daf  0.266667  4   15
scaffold_1  10000220  10000221  .  LE_2n  .  daf  0.400000  6   15
scaffold_1  10000236  10000237  .  LE_2n  .  daf  0.466667  7   15
```

The numerator column gives the raw derived allele count; the denominator gives the total called alleles. Both can be used directly to build a counts-based SFS for downstream tools (e.g., dadi, fastsimcoal2).

### Folded SFS (using MAF)

Use `maf` instead of `daf` for a folded SFS that does not require a polarized VCF:

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s maf,miss --persite \
  | piawka filt -e 'miss == 0' \
  | awk '$7=="maf" && $5=="LE_2n"' \
  | awk '{print $8}' | sort -n | uniq -c | head -10
```

## pi0 / pi4 — synonymous vs. 0-fold nucleotide diversity

A classic neutrality test compares nucleotide diversity at synonymous (4-fold degenerate) sites (π₄) to diversity at 0-fold degenerate sites (π₀). The ratio π₀/π₄ reflects purifying selection against nonsynonymous mutations.

### Using degenotate output

The `examples/degenotate/` folder contains degeneracy classification files generated by [degenotate](https://github.com/harvardinformatics/degenotate):

```console
$ ls examples/degenotate/
degeneracy-all-sites.bed   fourfolds.bed
piawka_fourfolds.tsv       piawka_zerofolds.tsv
zerofolds.bed
```

### Two-BED approach: pi by gene

```console
# pi at 4-fold sites, summarized per gene
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s pi \
              -T examples/degenotate/fourfolds.bed \
              -b genes.bed \
  > pi4_genes.bed

# pi at 0-fold sites, summarized per gene
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s pi \
              -T examples/degenotate/zerofolds.bed \
              -b genes.bed \
  > pi0_genes.bed

# Inspect pi4 results
$ head -4 pi4_genes.bed | column -t
#chr        start     end       locus      pop1      pop2  stat  value       numerator  denominator
scaffold_1  10035093  10035276  AL5G20950  LE_2n     .     pi    0.00341206  62         18170
scaffold_1  10035093  10035276  AL5G20950  PUWS_4n   .     pi    0.00327869  60         18300
scaffold_1  10035093  10035276  AL5G20950  SibA_2n   .     pi    0.00209790  38         18126
```

### Alternative: piawka sum -b

If pi was already calculated over the whole region (e.g., exon-level BED), `piawka sum -b` can re-aggregate results by the 4th column (locus name) of a gene-level BED:

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s pi \
              -T examples/degenotate/fourfolds.bed \
  | piawka sum -b genes.bed > pi4_by_gene.bed
```

## Distance matrix

A pairwise distance matrix is essential for phylogenetic and clustering analysis. Use `piawka dist` to convert `piawka calc` output into PHYLIP or NEXUS format.

### PHYLIP matrix (default)

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -b genes.bed -g groups.tsv -s dxy \
  | piawka dist - | column -t
5
CESiberia_2n  0.000000  0.009476  0.010348  0.009248  0.015265
CESiberia_4n  0.009476  0.000000  0.009521  0.006877  0.014528
LE_2n         0.010348  0.009521  0.000000  0.009551  0.015448
PUWS_4n       0.009248  0.006877  0.009551  0.000000  0.014237
SibA_2n       0.015265  0.014528  0.015448  0.014237  0.000000
```

### NEXUS matrix (for SplitsTree / SpecTRE)

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -b genes.bed -g groups.tsv -s dxy \
  | piawka dist --nexus - > distances.nex

# Open distances.nex in SplitsTree4 for network visualisation
```

### Using Fst as a distance

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -b genes.bed -g groups.tsv -s fst \
  | piawka dist -s fst - | column -t
```

> **Note**: Fst is not a proper distance metric (it does not satisfy the triangle inequality), but it can still be useful for exploratory clustering.

## Heterozygosity and within-individual diversity

Within-individual heterozygosity is equivalent to nucleotide diversity measured within a single diploid (or polyploid) individual — that is, the probability that the two alleles at a heterozygous site differ.

### Per-individual heterozygosity (divide mode)

Use `--groups divide` to treat each sample as its own group:

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -b genes.bed -g divide -s pi \
  | head -4 | column -t
#chr        start     end       locus      pop1      pop2  stat  value      numerator  denominator
scaffold_1  10035093  10035276  AL5G20950  al1_1_4n  .     pi    0.006250   1          160
scaffold_1  10035093  10035276  AL5G20950  al2_1_4n  .     pi    0.043750   7          160
scaffold_1  10035093  10035276  AL5G20950  al3_1_4n  .     pi    0.006250   1          160
```

### Average heterozygosity per group

To compute the average heterozygosity across all individuals in a group, pipe the per-individual output to `piawka sum -g`:

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -b genes.bed -g divide -s pi \
  | piawka sum -g groups.tsv \
  | head -4 | column -t
#chr        start     end       locus      pop1          pop2  stat  value       numerator  denominator
scaffold_1  10035093  10035276  AL5G20950  CESiberia_2n  .     pi    0.015625   50          3200
scaffold_1  10035093  10035276  AL5G20950  CESiberia_4n  .     pi    0.015875   254         16000
scaffold_1  10035093  10035276  AL5G20950  LE_2n         .     pi    0.013542   65          4800
```

## Segregating and differentially fixed sites

`pi` and `dxy` together classify sites into biologically meaningful categories:

| pi_i | pi_j | dxy | Interpretation |
|------|------|-----|----------------|
| > 0  | = 0  | > 0 | Privately segregating in group i |
| = 0  | > 0  | > 0 | Privately segregating in group j |
| = 0  | = 0  | > 0 | Fixed difference between groups |
| > 0  | > 0  | > 0 | Shared polymorphism (or partially shared) |
| = 0  | = 0  | = 0 | Monomorphic in both groups |

Use `piawka filt` to extract each category from `--persite` output.

### Fixed differences (differentially fixed sites)

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s pi,dxy --persite \
  | piawka filt -e 'pi == 0 && dxy > 0' \
  | head -4 | column -t
```

### Privately segregating sites in one group

```console
$ piawka calc -v alyrata_scaff_1_10000k-10500k.vcf.gz \
              -g groups.tsv -s pi,dxy --persite \
  | piawka filt -e 'pop1 == "LE_2n" && pi > 0 && dxy > 0' \
  | head -4 | column -t
```

### Shared polymorphisms — a caveat

Sites with `pi > 0` in both groups and `dxy > 0` appear to be shared polymorphisms, but this classification is unreliable for **multiallelic sites**:

```
Example site:  Group A  →  T/A  (pi > 0)
               Group B  →  T/G  (pi > 0)
               dxy for T vs. A and T vs. G → dxy > 0
```

Here both groups are polymorphic and dxy > 0, but the groups share the same REF allele T while carrying *different* ALT alleles — not a shared polymorphism in the biological sense. Filter out multiallelic sites with `piawka filt -e 'pop2 != "." || dxy > 0'` or run `piawka calc` without `--mult` (the default) to restrict to biallelic comparisons within each group pair.

When the VCF is large and piawka applies per-group biallelic filtering, some of these artefacts are already removed. Still, interpret shared polymorphisms cautiously whenever the source VCF includes sites with multiple segregating alleles across populations.
